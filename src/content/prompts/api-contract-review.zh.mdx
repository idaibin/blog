这个 Prompt 用于前后端接口联调或变更审查。它不做泛化的代码质量点评，而是沿着真实调用链确认路径、方法、参数、类型、响应字段和页面行为是否一致。

## 使用场景

- 后端接口和前端页面同时发生了修改；
- 页面请求失败，但接口名称看起来没有问题；
- 提交前需要确认接口契约没有遗漏；
- 旧字段、Mock 或临时兼容可能仍然残留。

## 可复制 Prompt

```text
请对当前范围内的前后端 API 契约进行只读审查。不要先修改代码，也不要用函数名或文件名推测接口已经对齐。

首先读取当前路径下生效的 AGENTS.md、接口文档和相关代码，并确认 Git 工作区状态。然后按接口逐个追踪以下完整链路：

1. 后端路由和真实 URL
2. HTTP method
3. path、query 和 body 参数
4. 参数是否必填、默认值和数据转换
5. 响应中的 code、message/msg、data、分页和错误字段
6. 后端错误码与错误处理
7. 前端 API 封装的路径、method 和参数映射
8. TypeScript 类型定义
9. 页面字段读取、提交和提交前转换
10. loading、empty、error 和空值状态
11. 旧接口、旧字段、Mock、临时兼容和残留代码

每个问题必须包含：

- 严重程度
- 后端文件路径和位置
- 前端文件路径和位置
- 具体不一致点
- 真实影响
- 最小修复建议

最后输出：

| 接口 | 后端状态 | 前端 API 状态 | 页面调用状态 | 问题 | 验证方式 |
| --- | --- | --- | --- | --- | --- |

如果证据不足，明确写“未验证”并说明缺少什么。不要把静态阅读当成运行成功，也不要在没有请求、日志、测试或页面证据时宣称接口已经可用。
```

## 修复边界

如果后续要求修复，只修改已经确认的契约问题。修复后根据项目实际能力运行 typecheck、测试、构建、接口请求或页面验证，并分别说明哪些链路已经验证、哪些仍有风险。
