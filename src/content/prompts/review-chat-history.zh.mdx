这个 Prompt 不只是总结聊天内容。它更关注对话中已经形成的决定、可复用经验、工作阻力和仍未闭合的问题，适合日终复盘或阶段性回顾。

它只能使用实际可访问的记录。如果当前环境看不到完整历史，就必须说明覆盖范围，不能假装已经读过全部对话。

## 可复制 Prompt

```text
请复盘指定日期或时间范围内实际可访问的对话记录，输出简洁、技术化、面向后续行动的工作总结。

开始时先说明：

- 实际读取了哪些对话或时间范围
- 哪些历史不可访问
- 是否存在信息缺口

只把对话中明确出现的路径、提交、验证结果、完成状态和决策当作事实。不要推断未看到的工作。重复讨论只保留最新结论，并区分“明确决定”和“基于多次对话形成的观察”。

输出结构：

1. 核心方向
   - 这些对话实际围绕什么目标展开
2. 进展与决策
   - 已完成内容、确认的取舍和经过验证的结果
3. 可沉淀经验
   - 可以复用于以后任务的规则、判断或工作方式
4. 阻力与噪音
   - 重复沟通、需求漂移、无效尝试和可以避免的往返
5. 未闭合事项
   - 未完成工作、待确认选择和明确阻塞
6. 下一步
   - 1–3 个具体且高价值的行动
7. 长期信号
   - 一条明确标记为“观察”的长期趋势，不把推断写成事实

如果某一部分没有足够证据，写“没有可确认内容”。避免流水账、泛泛鼓励和虚构的效率评价。
```

## 使用建议

日终任务可以把日期、时区和可访问的会话范围作为输入。阶段复盘则应同时提供相关任务、文档或 Git 证据，避免只根据聊天措辞判断真实完成状态。
