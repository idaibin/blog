这个 Prompt 把之前分散的每日 AI 摘要、研究搜索和可复用资产发现合并成一个可配置任务。主题可以变化，但来源验证、去重和证据规则保持稳定。

## 配置

```text
关注主题：[产品、仓库、公司、技术或问题]
使用目的：[新闻跟踪 / 工程决策 / 资产发现 / 其他]
执行计划：[默认每天 09:00]
时区：[默认 Asia/Shanghai]
回看窗口：[默认 72 小时]
最多条数：[默认 8]
已有本地清单：[可选，填写索引、仓库或目录]
输出语言：[默认使用用户语言]
```

## 可复制 Prompt

```text
请根据以上配置搜索当前的一手来源，生成一份定时主题摘要。

只选择会影响指定使用目的的更新。优先使用官方文档、更新日志、论文、技术报告、原始仓库和维护者工程文章。只有缺少一手来源或确实需要独立背景时，才使用可靠的二手报道。

执行规则：

1. 打开并核实每个入选来源，不使用搜索结果摘要代替原文。
2. 优先记录事件或版本实际发生的日期，而不只是文章发布日期。
3. 按事件、版本、仓库和规范化 URL 去重，保留最直接的一手来源。
4. 排除纯营销、重复转载、无明确工程影响的泛化总结，以及没有标记依据的传闻。
5. 区分正式发布、研究结果、预览、推断和猜测。
6. 如果提供了本地清单，先检查是否已经存在同类资产，再建议新增、扩展或不处理。
7. 如果没有可靠更新，返回“没有重要更新”，不要用低质量条目凑数。
8. 如果不能访问网络或无法验证当前信息，停止并说明限制。

开头说明覆盖时间、关注主题、检查的来源范围和限制。

每条内容包含：

- 标题、类型、状态和事件日期
- 一手来源名称和直接链接
- 简短事实摘要
- 与使用目的的关系
- 建议行动，或“不处理”

结尾输出：

1. 关键结论：最多三条跨项目判断
2. 值得验证：需要进一步实验或核实的主张、工具和决定
3. 可沉淀资产：适合变成 Prompt、Skill、工作流或参考资料的候选；没有则写“不处理”

不要编造引用、日期、性能数据、可用性、本地重复情况或验证结果。不要大段复制来源内容。
```

## 为什么只保留一个版本

“每日 AI 摘要”和“搜索可复用 AI 资产”使用的是同一组研究与筛选能力。把关注主题和使用目的变成配置，比继续维护多个只有主题不同的 Prompt 更容易长期更新。
