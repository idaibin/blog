### <a href="https://www.reuters.com/science/anthropic-unveils-claude-science-ai-platform-scientific-research-2026-06-30/" target="_blank" rel="noopener noreferrer">Anthropic launches Claude Science for research workflows ↗</a>

Reuters · 2026-06-30 · product release / scientific AI

Reuters reports that Anthropic launched Claude Science, an AI research workbench for scientists that is designed to support research workflows, data analysis, and complex computing workflow management.

**Why it matters**: AI products are moving from general chat surfaces into vertical workbenches where data handling, compute orchestration, and auditability matter.

### <a href="https://arxiv.org/abs/2606.30639" target="_blank" rel="noopener noreferrer">Self-evolving world models improve LLM agent planning ↗</a>

arXiv · 2026-06-29 · research / agent planning

The paper introduces WorldEvolver, a framework that revises a world model's deployment-time context without updating the downstream agent or model parameters, using episodic memory, semantic memory, and selective foresight; experiments cover ALFWorld, ScienceWorld, and AgentBoard-style planning evaluations.

**Why it matters**: Long-horizon agent performance can improve through test-time memory revision, failure-derived rules, and confidence filtering rather than only through larger models.

### <a href="https://arxiv.org/abs/2606.30555" target="_blank" rel="noopener noreferrer">Linguistic Firewall tests agents before routing tasks ↗</a>

arXiv · 2026-06-29 · research / multi-agent security

This AIWILD / ICML 2026 workshop paper proposes ANTAP, a routing architecture that avoids agent self-descriptions and static representations, actively tests agent capabilities, and reports near-zero ASR against description-based injection attacks.

**Why it matters**: Multi-agent orchestration needs routing based on verified behavior, not on untrusted metadata or agent-written claims about capability.

### <a href="https://arxiv.org/abs/2606.30531" target="_blank" rel="noopener noreferrer">Entity binding failures expose a tool-use safety gap ↗</a>

arXiv · 2026-06-29 · research / tool-use safety

The paper separates tool correctness from entity correctness: an agent may call the right tool while acting on the wrong contact, document, thread, or customer account; in a 60-task diagnostic evaluation, action-oriented baselines had no wrong-tool errors but still produced wrong-entity actions in 24.0% to 26.0% of runs.

**Why it matters**: Enterprise agents need entity resolution, ambiguity gates, provenance tracking, and action preconditions in addition to API-call validation.

### <a href="https://arxiv.org/abs/2606.30383" target="_blank" rel="noopener noreferrer">Principal loyalty becomes a benchmarkable agent alignment problem ↗</a>

arXiv · 2026-06-29 · research / agent alignment

The paper studies principal loyalty in multi-party agents: an agent acts for one principal while also communicating with a counterparty whose interests may diverge; the authors introduce PrincipalBench and report a strong trade-off between leak prevention and over-refusal.

**Why it matters**: Enterprise agents that negotiate, screen requests, or mediate between parties need explicit policies for whose interests they represent.
