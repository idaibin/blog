### <a href="https://www.reuters.com/science/anthropic-unveils-claude-science-ai-platform-scientific-research-2026-06-30/" target="_blank" rel="noopener noreferrer">Anthropic launches Claude Science for research workflows ↗</a>

Reuters · 2026-06-30 · product release / scientific AI

Reuters 报道，Anthropic 推出 Claude Science，这是一个面向科学家的 AI 研究工作台，用于研究流程、数据分析和复杂计算工作流管理。

**值得关注**：AI 产品正在从通用聊天界面继续下沉到垂直工作台，科学研究场景会更重视数据、计算和审计链路。

### <a href="https://arxiv.org/abs/2606.30639" target="_blank" rel="noopener noreferrer">Self-evolving world models improve LLM agent planning ↗</a>

arXiv · 2026-06-29 · research / agent planning

这篇论文提出 WorldEvolver，在不更新下游 Agent 和模型参数的情况下，用 episodic memory、semantic memory 和 selective foresight 在部署期修正世界模型上下文，并在 ALFWorld、ScienceWorld 和 AgentBoard 相关评测中报告了更好的预测和规划表现。

**值得关注**：长任务 Agent 的改进不只来自更强模型，也可以来自测试时记忆修正、失败反馈沉淀和低置信预测过滤。

### <a href="https://arxiv.org/abs/2606.30555" target="_blank" rel="noopener noreferrer">Linguistic Firewall tests agents before routing tasks ↗</a>

arXiv · 2026-06-29 · research / multi-agent security

这篇 AIWILD / ICML 2026 workshop 论文提出 ANTAP，不再依赖 Agent 自我描述或静态表示来做多 Agent 路由，而是通过主动能力测试形成非文本行为表示，并报告其对描述注入攻击的 ASR 接近零。

**值得关注**：多 Agent 系统的路由安全需要从“相信 Agent 怎么描述自己”转向“用可验证行为决定谁能接任务”。

### <a href="https://arxiv.org/abs/2606.30531" target="_blank" rel="noopener noreferrer">Entity binding failures expose a tool-use safety gap ↗</a>

arXiv · 2026-06-29 · research / tool-use safety

这篇论文把 tool correctness 和 entity correctness 分开讨论：Agent 可以选对工具，却操作错联系人、文档、线程或客户账户；在 60 个任务的诊断评估中，动作型基线没有选错工具，但仍出现 24.0% 到 26.0% 的 wrong-entity actions。

**值得关注**：真实企业 Agent 不能只校验“调用了正确 API”，还需要实体解析、歧义确认、来源追踪和高风险动作前置条件。

### <a href="https://arxiv.org/abs/2606.30383" target="_blank" rel="noopener noreferrer">Principal loyalty becomes a benchmarkable agent alignment problem ↗</a>

arXiv · 2026-06-29 · research / agent alignment

这篇论文研究多方交互里的 principal loyalty：Agent 一边代表委托方执行任务，一边和利益可能不同的对手方交流；作者提出 PrincipalBench，并观察到泄露风险和过度拒绝之间存在明显权衡。

**值得关注**：企业 Agent 会越来越多地代表某个主体谈判、筛选和协调，安全策略必须明确“Agent 到底站在哪一方”。
