### <a href="https://www.businessinsider.com/coinbase-ceo-brian-armstrong-low-ai-spend-maintain-token-usage-2026-6" target="_blank" rel="noopener noreferrer">Coinbase is treating AI spend as infrastructure, not a token cap problem ↗</a>

Business Insider · 2026-06-29 · industry practice / AI cost control

Business Insider reported that Coinbase CEO Brian Armstrong outlined ways to lower AI costs without suppressing engineer token usage: cheaper default models, automated model routing, better caching, leaner context, and shared cost visibility.

**Why it matters**: Internal AI adoption is moving from blunt usage limits toward LLM gateways, cost observability, and routing infrastructure.

### <a href="https://www.tomshardware.com/tech-industry/cyber-security/ai-coding-agents-can-be-tricked-into-installing-malware-via-clean-github-repositories-mozillas-0din-team-shows-how-claude-code-can-be-exploited-by-its-own-helpfulness" target="_blank" rel="noopener noreferrer">Mozilla 0din report highlights trust risks in AI coding agents ↗</a>

Tom's Hardware · 2026-06-29 · security report / coding agents

Tom's Hardware reported that Mozilla's 0din team demonstrated how AI coding agents can be steered by a seemingly normal repository and setup instructions into unsafe execution; this is a concrete agent-security report, not a product release.

**Why it matters**: Coding-agent safety needs repository trust boundaries, command approval, sandboxing, and auditable execution instead of relying on a model's helpfulness.

### <a href="https://arxiv.org/abs/2606.28270" target="_blank" rel="noopener noreferrer">Agent-Native Immune System frames runtime defense inside the agent loop ↗</a>

arXiv · 2026-06-29 · research / agent security

This paper proposes an Agent-Native Immune System that moves agent defense from perimeter controls and training-time alignment into the agent's runtime cognitive loop, with attention to memory poisoning, tool-chain manipulation, and multi-agent protocol attacks.

**Why it matters**: Agent security is shifting from prompt constraints toward runtime immunity, isolation layers, continual defensive learning, and standardized evaluation metrics.

### <a href="https://arxiv.org/abs/2606.27814" target="_blank" rel="noopener noreferrer">ATOD combines distillation and RL for multi-turn autonomous agents ↗</a>

arXiv · 2026-06-29 · research / agent training

ATOD proposes a training method for multi-turn autonomous agents: on-policy distillation dominates early learning to approach teacher behavior, then reinforcement learning is strengthened to improve long-horizon task rewards; the paper reports higher success rates on ALFWorld, WebShop, and Search-QA.

**Why it matters**: Small-agent post-training is moving toward hybrid “imitate first, explore later” recipes that may reduce deployment cost while improving long-task execution.

### <a href="https://arxiv.org/abs/2606.28235" target="_blank" rel="noopener noreferrer">Govern the Repository, Not the Agent measures ecosystem-level AI coding risk ↗</a>

arXiv · 2026-06-29 · research / AI software engineering

This software-engineering paper analyzes more than 930,000 agent-authored pull requests and argues that much integration friction belongs to the repository ecosystem rather than to any single agent or isolated contribution.

**Why it matters**: AI-native software governance should measure repository-level integration risk, process maturity, and maintenance cost, not only an agent's task pass rate.
